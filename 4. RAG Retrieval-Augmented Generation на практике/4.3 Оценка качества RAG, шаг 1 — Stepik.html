Оценка качества RAG

Почему RAG нужно оценивать?

Без оценки вы не знаете:

  • Находит ли система нужный фрагмент?
  • Не выдумывает ли LLM ответ, которого нет в контексте?
  • Достаточно ли контекста для ответа?

 Это как тестировать API без unit-тестов: «работает — не работает» — недостаточно.

 

Ключевые метрики RAG

1. Faithfulness — верность контексту

Не врёт ли LLM?
Оценивает, насколько ответ основан только на предоставленном контексте, а не на внутренних знаниях модели.
Низкий балл = галлюцинации («Alembic поддерживает —undo» — но в документации об этом нет).

2. ContextRelevance — релевантность найденного фрагмента

Находит ли система нужное?
Оценивает, насколько извлечённый фрагмент полезен для ответа на вопрос.
Низкий балл = векторный поиск вернул про Docker, а вопрос — про миграции.

3. AnswerRelevancy — релевантность ответа

Отвечает ли LLM по делу?
Измеряет семантическое сходство между вопросом и ответом через embedding-модель.
Низкий балл — нормально, если:

  • формулировка ответа сильно отличается от вопроса,
  • используется слабая embedding-модель (all-MiniLM-L6-v2),
  • ответ технически верен, но «не в том стиле».

 Faithfulness и ContextRelevance — надёжны даже локально.
AnswerRelevancy — чувствителен к формулировке и embedding-модели. Не гонитесь за 1.0.

 Чаще всего используют первые три метрики: Context Relevancy, Faithfulness, Answer Relevancy.

 

RAGAS

RAGAS (Retrieval-Augmented Generation Assessment) — библиотека с открытым исходным кодом для автоматической оценки качества RAG-пайплайнов.

  • Поддерживает локальные и облачные LLM (включая mistral, llama3 через Ollama).
  • Не оценивает саму LLM, а оценивает работу RAG-пайплайна в целом.

Установка:

pip install ragas

 

Оценка RAG с RAGAS

Шаг 1. Установка зависимостей

pip install ragas openai sentence-transformers

 

Шаг 2. Подготовка данных

В RAGAS 0.4+ используется специальный формат данных:

from ragas.dataset_schema import SingleTurnSample, EvaluationDataset

dataset = EvaluationDataset(samples=[
    SingleTurnSample(
        user_input="Какой инструмент используется для миграций базы данных?",
        response="Для миграций базы данных используется Alembic.",
        retrieved_contexts=["Alembic используется для управления миграциями базы данных в SQLAlchemy."]
    ),
    SingleTurnSample(
        user_input="Что такое FastAPI?",
        response="FastAPI — это современный Python-фреймворк для создания API.",
        retrieved_contexts=["FastAPI — это современный Python-фреймворк для создания API."]
    )
])

 

Шаг 3. Настройка LLM и embedding-модели

import asyncio
from openai import AsyncOpenAI
from ragas.llms import llm_factory
from ragas.embeddings import HuggingFaceEmbeddings

# LLM через Ollama (OpenAI-совместимый API)
client = AsyncOpenAI(base_url="http://localhost:11434/v1", api_key="ollama")
llm = llm_factory("mistral", client=client)

# Локальная embedding-модель
embeddings = HuggingFaceEmbeddings(model="all-MiniLM-L6-v2")

Убедитесь, что модель (mistral, llama3 и т.д.) установлена в Ollama

 

Шаг 4. Оценка

RAGAS оценивает качество RAG-системы через три основные метрики, каждая из которых — отдельный класс:

1. Faithfulness

2. ContextRelevance

3. AnswerRelevancy

 Все три метрики возвращают число от 0 до 1 (чем ближе к 1 — тем лучше).
Используются одновременно, чтобы оценить RAG-пайплайн целиком.

from pydantic import BaseModel
from ragas.metrics.collections import Faithfulness, AnswerRelevancy, ContextRelevance

class Result(BaseModel):
    faithfulness: float
    answer_relevancy: float
    context_relevance: float

async def evaluate_sample(sample) -> Result:
    faith = await Faithfulness(llm=llm).ascore(
        user_input=sample.user_input,
        response=sample.response,
        retrieved_contexts=sample.retrieved_contexts
    )
    ar = await AnswerRelevancy(llm=llm, embeddings=embeddings).ascore(
        user_input=sample.user_input,
        response=sample.response
    )
    cr = await ContextRelevance(llm=llm, embeddings=embeddings).ascore(
        user_input=sample.user_input,
        retrieved_contexts=sample.retrieved_contexts
    )
    return Result(
        faithfulness=faith.value,
        answer_relevancy=ar.value,
        context_relevance=cr.value
    )

async def main():
    tasks = [evaluate_sample(sample) for sample in dataset.samples]
    results = await asyncio.gather(*tasks)
    for i, res in enumerate(results):
        print(f"Sample {i+1}: {res}")

if __name__ == "__main__":
    asyncio.run(main())

 

Запуск

 Запустите у себя:

import asyncio
from openai import AsyncOpenAI
from pydantic import BaseModel
from ragas.llms import llm_factory
from ragas.embeddings import HuggingFaceEmbeddings
from ragas.dataset_schema import SingleTurnSample, EvaluationDataset
from ragas.metrics.collections import Faithfulness, AnswerRelevancy, ContextRelevance

# === Датасет ===
dataset = EvaluationDataset(samples=[
    SingleTurnSample(
        user_input="Какой инструмент используется для миграций базы данных?",
        response="Для миграций базы данных используется Alembic.",
        retrieved_contexts=["Alembic используется для управления миграциями базы данных в SQLAlchemy."]
    ),
    SingleTurnSample(
        user_input="Что такое FastAPI?",
        response="FastAPI — это современный Python-фреймворк для создания API.",
        retrieved_contexts=["FastAPI — это современный Python-фреймворк для создания API."]
    )
])

# === LLM и Embeddings ===
client = AsyncOpenAI(base_url="http://localhost:11434/v1", api_key="ollama")
llm = llm_factory("mistral", client=client)
embeddings = HuggingFaceEmbeddings(model="all-MiniLM-L6-v2")

# === Модель результата ===
class Result(BaseModel):
    faithfulness: float
    answer_relevancy: float
    context_relevance: float

# === Оценка одной строки ===
async def evaluate_sample(sample: SingleTurnSample) -> Result:
    faith = await Faithfulness(llm=llm).ascore(
        user_input=sample.user_input,
        response=sample.response,
        retrieved_contexts=sample.retrieved_contexts
    )
    ar = await AnswerRelevancy(llm=llm, embeddings=embeddings).ascore(
        user_input=sample.user_input,
        response=sample.response
    )
    cr = await ContextRelevance(llm=llm, embeddings=embeddings).ascore(
        user_input=sample.user_input,
        retrieved_contexts=sample.retrieved_contexts
    )
    return Result(
        faithfulness=faith.value,
        answer_relevancy=ar.value,
        context_relevance=cr.value
    )

# === Запуск ===
async def main():
    tasks = [evaluate_sample(sample) for sample in dataset.samples]
    results = await asyncio.gather(*tasks)
    for i, res in enumerate(results):
        print(f"Sample {i+1}: {res}")

if __name__ == "__main__":
    asyncio.run(main())

 Пример вывода:

Sample 1: faithfulness=1.0 answer_relevancy=0.0457 context_relevance=1.0
Sample 2: faithfulness=1.0 answer_relevancy=0.4654 context_relevance=1.0

 

Почему answer_relevancy может быть низким?

  • Embedding-модель (all-MiniLM-L6-v2) слабо улавливает технические нюансы,
  • Формулировка ответа отличается от вопроса («инструмент» ↔ «используется»),
  • RAGAS сравнивает векторы, а не логику — даже правильный ответ может быть «далек» в embedding-пространстве.

 Не паникуйте из-за низкого answer_relevancy.
Сравнивайте относительные изменения:
— «чанкинг с overlap» vs «по абзацам» → растёт context_relevance,
— «без reranking» vs «с FlashRank» → растёт faithfulness.

Советы

  1. Интегрируйте RAGAS в CI/CD: запускайте оценку при каждом изменении пайплайна,
  2. Используйте одну и ту же LLM и embedding-модель для всех тестов — иначе сравнение бессмысленно,
  3. Не используйте phi3 или gemma4 для оценки — они дают шумные метрики. Лучше mistral, llama3:8b,
  4. Сравнивайте версии, а не абсолютные цифры — «стало лучше или хуже?» важнее, чем «0.9 или 0.95».
